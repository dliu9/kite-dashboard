Use Case:   Alert traders to high-confidence event-driven volatility periods
            ("Event clustering score hit 78 today - expect elevated volatility")
Business Value:
  - Identifies when concentrated event activity creates price momentum
  - Separates noise (single events) from signal (event cascades)
  - Enables position sizing by cluster confidence
  - Helps time entry/exit around event windows

────────────────────────────────────────────────────────────────────
Definition: Weighted score of predictive power per event type using 21-day

Calculation: 21-day rolling calculation per event type; recalculates daily
Use Case:   Dynamic portfolio weighting and threshold adjustment
            ("Milestone events are now 78/100 reliable; increase position weight")

  - Data-driven signal confidence for risk management
  - Enables A/B testing of trading strategies against threshold levels

4. SQL QUERIES FOR NEW KPI IMPLEMENTATION

QUERY 1: Event Clustering Score (7-Day Rolling, for Dashboard)
────────────────────────────────────────────────────────────────────

SELECT 
  DATE(e.date) as event_date,
  COUNT(DISTINCT e.id) as event_count_7d,
  ROUND(AVG(CASE 
    WHEN e.sentiment_label = 'positive' THEN 1
    WHEN e.sentiment_label = 'neutral' THEN 0
    WHEN e.sentiment_label = 'negative' THEN -1
    ELSE 0 END), 2) as avg_sentiment,
  ROUND(
    (MAX(p.price_usd) - MIN(p.price_usd)) / MIN(p.price_usd) * 100, 
    2
  ) as vol_spike_pct_7d,
  ROUND(
    COUNT(DISTINCT e.id) * 
    ROUND(AVG(CASE 
      WHEN e.sentiment_label = 'positive' THEN 1
      WHEN e.sentiment_label = 'neutral' THEN 0
      WHEN e.sentiment_label = 'negative' THEN -1
      ELSE 0 END), 2) *
    (ROUND(
      (MAX(p.price_usd) - MIN(p.price_usd)) / MIN(p.price_usd) * 100, 
      2
    ) / 100),
    2
  ) as clustering_score
FROM events e
INNER JOIN price_history p ON DATE(p.date) >= DATE(e.date, '-6 days') 
                              AND DATE(p.date) <= e.date
WHERE e.date IS NOT NULL
GROUP BY DATE(e.date)
ORDER BY event_date DESC
LIMIT 30;

Refresh: Daily (execute at 09:00 UTC)
Caching: Cache results for 24 hours


QUERY 2: Event Reliability Index by Type (21-Day Rolling)
────────────────────────────────────────────────────────────────────

WITH event_windows AS (
  SELECT 
    e.event_type,
    DATE(e.date) as event_date,
    CASE WHEN p2.price_usd > p1.price_usd THEN 1 ELSE 0 END as positive_move,
    CASE WHEN p2.price_usd IS NOT NULL 
         THEN ((p2.price_usd - p1.price_usd) / p1.price_usd * 100) 
         ELSE 0 END as return_7d
  FROM events e
  LEFT JOIN price_history p1 ON DATE(e.date) = p1.date
  LEFT JOIN price_history p2 ON DATE(e.date, '+7 days') = p2.date
  WHERE e.date IS NOT NULL
)
SELECT 
  event_type,
  DATE(event_date) as calc_date,
  COUNT(*) as event_count,
  ROUND(
    (SUM(positive_move) * 100.0 / NULLIF(COUNT(*), 0)) * 0.4 +
    AVG(return_7d) * 0.3 +
    (COUNT(DISTINCT event_date) / 21.0) * 0.3,
    2
  ) as reliability_index
FROM event_windows
WHERE DATE(event_date) >= DATE('now', '-21 days')
GROUP BY event_type, DATE(event_date)
ORDER BY event_type, event_date DESC;

Refresh: Daily (execute at 09:00 UTC)
Caching: Cache results for 24 hours
Note: Recompute when new events added or price data backfilled


QUERY 3: Data Completeness Score (Per Event)
────────────────────────────────────────────────────────────────────

SELECT 
  id,
  event_type,
  date,
  datetime_str,
  sentiment_label,
  ROUND(
    CASE WHEN date IS NOT NULL AND date != '' THEN 1 ELSE 0 END * 0.3 +
    CASE WHEN datetime_str IS NOT NULL AND datetime_str != '' THEN 1 ELSE 0 END * 0.3 +
    CASE WHEN sentiment_label IS NOT NULL THEN 1 ELSE 0 END * 0.2 +
    CASE WHEN date >= '2025-12-23' THEN 1 ELSE 0 END * 0.2,
    2
  ) as data_completeness_score
FROM events
ORDER BY data_completeness_score DESC, date DESC;

Refresh: On event creation; re-run when backfill completes
Caching: Can be materialized as view or computed column
Usage: Join to events table in all event-based queries with WHERE completeness >= 0.7
